What Run 1 Asked

Primary Question

Could a fast local model on this host deliver enough throughput and answer quality to justify a local-first workflow at all?

Models Compared

Qwen2.5-7B-Instruct-4bit, Qwen2.5-32B-Instruct-4bit, and Qwen2.5-Coder-32B-Instruct-4bit.

Task Shape

Throughput, a coding/repair task, and an architecture reasoning task. This was still a baseline run rather than a refined routing benchmark.

What It Found

Fast Local Already Mattered

The 7B lane was not merely acceptable. It won overall, led on throughput, and also led on the benchmarked coding and architecture tasks in this run.

No Immediate Need For Larger Default

Run 1 did not support paying the latency cost of a larger model as the default on this node.

Single-Model Default Emerged

The early recommendation was straightforward: if this node needs one practical local default, start with Qwen2.5-7B-Instruct-4bit.

Why It Still Matters

Run 1 is the foundation for everything that follows. The later routing story only matters because this run first showed that a small fast local lane was already credible on real tasks.
