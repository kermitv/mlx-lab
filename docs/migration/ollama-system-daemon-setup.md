# System Architecture: Managing the Ollama Launch Daemon

This guide describes the professional-grade method for managing the Ollama inference engine on macOS using `launchd`. By using a Launch Daemon, we ensure the engine is a persistent, system-level service that is resilient to reboots and easy to manage.

## Homebrew Install Note

When `brew install ollama` completes on macOS, Homebrew prints the following service-start guidance:

```bash
To start ollama now and restart at login:
  brew services start ollama
Or, if you don't want/need a background service you can just run:
  OLLAMA_FLASH_ATTENTION="1" OLLAMA_KV_CACHE_TYPE="q8_0" /opt/homebrew/opt/ollama/bin/ollama serve
```

Recorded host-specific choice:

- On a MacBook Pro M1 setup, the selected startup path was `brew services start ollama`.

Observed result on that machine:

```bash
kermitv@Kermits-MacBook-Pro ~ % brew services start ollama
==> Successfully started `ollama` (label: homebrew.mxcl.ollama)
kermitv@Kermits-MacBook-Pro ~ %
```

Implication:

- For that machine, Ollama was intentionally run as a Homebrew-managed background service rather than as a foreground `ollama serve` process.

## 🏗️ The Architectural Blueprint

The architecture consists of three layers:
1.  **The Wrapper Script (`~/scripts/start_ollama_server.sh`):** Contains the "God-Mode" configuration (Flash Attention, KV Cache tuning, etc.) and the actual `ollama serve` command.
2.  **The Launch Daemon (`/Library/LaunchDaemons/com.kermit.ollama_server.plist`):** The instruction manual that tells macOS how to load and run the wrapper script at boot.
3.  **The System Controller (`launchctl`):** The macOS utility used to start, stop, and monitor the service.

## 🛠️ Step-by-Step Implementation

### 1. Create the Wrapper Script
First, we take the complex, environment-heavy command and encapsulate it into an executable script.

1.  Create the file:
    ```bash
    nano ~/scripts/start_ollama_server.sh
    ```
2.  **Paste the high-performance configuration:**
    ```bash 
    #!/bin/zsh
    export OLLAMA_FLASH_ATTENTION="1"
    export OLLAMA_KV_CACHE_TYPE="q8_0"
    export OLLAMA_NUM_PARALLEL="4"
    export OLL_MAX_LOADED_MODELS="3"
    export OLLAMA_HOST="0.0.0.0"
    export OLLAMA_ORIGINS="*"
    
    /opt/homebrew/opt/ollama/bin/ollama serve
    ```
3.  **Make it executable:**
    ```bash
    chmod +x ~/scripts/start_ollama_server.sh
    ```

### 2. Create the Launch Daemon (`.plist`)
Next, we create the `.plist` file that `launchd` uses to manage the service.

1.  Create the daemon file (requires `sudo`):
    ```bash
    sudo nano /Library/LaunchDaemons/com.kermit.ollama_server.plist
    ```
2.  **Paste the configuration:**
    *(Note: Replace `YOUR_USERNAME` with your actual macOS username, e.g., `kermitv`)*
    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
    <plist version="1.0">
    <dict>
        <key>Label</key>
        <string>com.kermit.ollama_server</string>

        <key>ProgramArguments</key>
        <array>
            <string>/Users/YOUR_USERNAME/scripts/start_ollama_server.sh</string>
        </array>

        <key>RunAtLoad</key>
        <true/>

        <key>KeepAlive</key>
        <true/>

        <key>StandardOutPath</key>
        <string>/tmp/ollama_server.log</string>

        <key>StandardErrorPath</key>
        <string>/tmp/ollama_server_err.log</string>
    </dict>
    </plist>
    ```
3.  **Set correct permissions:**
    ```bash
    sudo chown root:wheel /Library/LaunchDaemons/com._kermit.ollama_server.plist
    ```

### 3. Register and Start the Service
1.  **Load the service into the system:**
    ```bash
    sudo launchctl bootstrap system /Library/LaunchDaemons/com.kermit.ollama_server.plist
    ```

## 🕹️ Service Management (The "Two-State" Model)

To manage a `launchd` service, you must understand the two different "states" the service can be in:
1.  **The Running State (Active):** The process is alive in RAM and processing requests.
2.  **The Registered State (Loaded):** The `.plist` is "known" to the system and will start on boot.

### 🔄 Lifecycle Commands

| Action | Command | Result |
| :--- | :--- | :--- |
| **Stop and Disable** | `sudo launchctl bootout system [path_to_plist]` | Kills the process and removes it from the boot sequence. |
| **Start and Enable** | `sudo launchctl bootstrap system [path_to_plist]` | Loads the configuration and starts the service immediately. |
| **Emergency Stop** | `sudo kill [PID]` | Kills the process but keeps it active for the next reboot. |
| **Verify Status** | `sudo launchctl print system/com.kermit.ollama_server` | Confirms if the service is currently loaded and active. |

### 🔍 Debugging
If the server fails to start, check the error log generated by the daemon:
```bash
tail -f /tmp/ollama_server_err.log
```
