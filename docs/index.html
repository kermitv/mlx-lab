What This Work Is Actually For

The benchmark line started as a local viability check, then became a more useful question: when is fast local good enough, when should work escalate to a stronger local specialist, and when does frontier quality justify the additional cost, latency, and dependency?

Repo-first Evidence

The benchmark artifacts, raw responses, scoring tables, and execution traces stay in the repository. That keeps the evidence inspectable and versioned.

Pages-first Narrative

This site is intentionally lighter-weight. It surfaces the story across runs, shows the charts, and points back into the repo when you need proof rather than prose.

Bounded Claims

Current conclusions are useful for mlx-lab operations. They are not yet canonical runtime policy and they should not be read that way.

Run Progression

The first four runs form a coherent sequence. Each run narrowed the question and improved the quality of the routing interpretation rather than just rerunning the same benchmark with different models.

Run 1

Established baseline local viability and showed that the fast 7B lane was already useful on this node.

Run 2

Shifted from a one-winner framing toward routing by use case, separating operational default from stronger reasoning and coding roles.

Run 3

Sharpened local escalation thresholds by task class, even though frontier access was unavailable in that run.

Run 4

Tested the local-to-frontier escalation boundary directly and produced the strongest current routing story in this repo.

Current Routing Story

Fast Local Default

Qwen2.5-7B-Instruct-4bit remains the best unattended default for bounded work in this host context.

Local Coding Specialist

Qwen2.5-Coder-32B-Instruct-4bit justifies escalation for harder multi-file coding and repair tasks.

Local Reasoning Lane

Qwen2.5-32B-Instruct-4bit is the stronger local choice for contradiction-heavy reasoning and governance-style work.

Frontier Boundary

Frontier answered some tasks better, but Run 4 did not support a universal “always escalate” rule.
