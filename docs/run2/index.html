What Run 2 Asked

Primary Question

Was there enough evidence to move from a single default model toward routing by use case: default, coding, reasoning, and governance?

Models Compared

Qwen2.5-7B-Instruct-4bit, Qwen2.5-32B-Instruct-4bit, Qwen2.5-Coder-32B-Instruct-4bit, and DeepSeek-R1-Distill-Qwen-32B-4bit.

Task Shape

Throughput, coding/repair, reasoning, and governance. This run was much closer to an operator routing benchmark than Run 1.

What It Found

7B Stayed The Default

The fast 7B lane remained the best operational default and did not give up enough quality to justify a larger default in this run.

32B Instruct Claimed Reasoning

Qwen2.5-32B-Instruct-4bit became the strongest hard-problem and reasoning model, giving the first real sign of a stronger local reasoning lane.

Coding Was Mixed

Coding and repair did not yet show a decisive specialist winner. The 7B default and coder 32B both remained viable, while DeepSeek underperformed in this run.

Why It Still Matters

Run 2 introduced the central benchmark idea that survives through Run 4: routing by task class is more useful than trying to crown one universal winner.
